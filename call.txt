This document outlines the API calls and expected parameters for the application, based on the main entry point. All calls are expected to be part of a URL query string.

---

### Command: `addproduct`
Adds a new product to the system.

**Parameters:**
*   `command=addproduct`
*   `username=<email>`
*   `password=<password>`
*   `supplier=<supplier_name>`
*   `description=<product_description>`
*   `price=<retail_price>`
*   `barcode=<barcode_number>` (Optional. If `createbarcode=true` and this is empty, a new EAN-13 barcode will be generated.)
*   `createbarcode=<true|false>` (Optional. Defaults to `true`.)

---

### Command: `updatestock`

### Command: `initiate_payment` (NEW - June 2026)
Creates a Stripe PaymentIntent for processing payments.

**Parameters:**
*   `command=initiate_payment`
*   `username=<email>`
*   `password=<password>`
*   `order_id=<order_identifier>`
*   `total=<amount_in_GBP>`
*   `type=<web_order|till_sale>`

**Response:**
Returns JSON with:
*   `id` - PaymentIntent ID
*   `client_secret` - Client secret for Stripe
*   `amount` - Amount in pence
*   `currency` - Currency code (gbp)
*   `status` - Payment status

---

### Command: `predict_reorder_date` (NEW - June 2026 - Boudica AI Forecasting)
Predicts when a product should be reordered based on sales velocity and current stock using Boudica AI.

**Parameters:**
*   `command=predict_reorder_date`
*   `username=<email>`
*   `password=<password>`
*   `barcode=<barcode_number>`

**Response:**
Returns JSON with:
*   `item_data` - Current product and stock information
*   `reorder_prediction` - AI-generated prediction with:
    - `days_until_reorder` - Days until stock runs out
    - `recommended_quantity` - Recommended reorder quantity
    - `priority` - Priority level (urgent|normal|low)
    - `explanation` - Reasoning behind the prediction
